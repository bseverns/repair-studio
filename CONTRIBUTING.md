# Contributing

Thanks for helping repair the world a little.

## How to contribute

- Use issue templates to submit **Intake Tickets** or propose station cards.
- Make small, focused PRs. Include before/after photos or test outputs when
  applicable.
- Update docs and signage if your change alters user behavior.

## Development

- Markdown lint runs in CI. Keep line lengths humane; use headings and
  lists.
- Store binary manuals under `/manuals/*`. Avoid committing massive files;
  prefer links when >25MB.
